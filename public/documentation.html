Authentication Endpoints

POST /auth/register

Description: Register a new user account

Request Body
{ "name": "John Doe", "email": "john@example.com", "password": "SecurePassword123" }
Validation Rules
  • name: required - 2-100 characters
  • email: required - Valid email format
  • password: required - Minimum 6 characters
201 Created
{ "success": true, "data": { "id": "507f1f77bcf86cd799439011", "name": "John Doe", "email": "john@example.com", "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." } }
POST /auth/login

Description: Authenticate user and receive JWT token

Request Body
{ "email": "john@example.com", "password": "SecurePassword123" }
200 OK
{ "success": true, "data": { "id": "507f1f77bcf86cd799439011", "name": "John Doe", "email": "john@example.com", "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." } }
GET /auth/me

Description: Get current user profile (requires authentication)

Headers
Authorization: Bearer <jwt_token>
200 OK
{ "success": true, "data": { "_id": "507f1f77bcf86cd799439011", "name": "John Doe", "email": "john@example.com", "createdAt": "2025-01-15T10:30:00Z", "updatedAt": "2025-01-15T10:30:00Z" } }

User Management Endpoints

GET /users

Description: Get list of all users

200 OK
{ "success": true, "data": [ { "_id": "507f1f77bcf86cd799439011", "name": "John Doe", "email": "john@example.com", "createdAt": "2025-01-15T10:30:00Z" } ] }
POST /users

Description: Create a new user

Request Body
{ "name": "Jane Doe", "email": "jane@example.com", "password": "SecurePassword123" }
201 Created
{ "success": true, "data": { "_id": "507f1f77bcf86cd799439012", "name": "Jane Doe", "email": "jane@example.com", "createdAt": "2025-01-15T11:00:00Z" } }

HTTP Status Codes

Code Meaning Description
200 OK Request successful
201 Created Resource created successfully
400 Bad Request Invalid input or validation error
401 Unauthorized Missing or invalid authentication token
404 Not Found Resource or endpoint not found
409 Conflict Resource already exists (e.g., duplicate email)
500 Server Error Internal server error

Error Response Format

All errors follow a consistent format:

{ "success": false, "message": "Error description", "stack": "Only included in development mode" }
